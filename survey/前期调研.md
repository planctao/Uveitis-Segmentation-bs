# 前期调研

日期：2026-06-11

## 1. 原论文概述

已阅读论文：[温医大-靳雨-baseline.pdf](/Users/tzh/CS/yjsbs/温医大-靳雨-baseline.pdf)

论文题目为 **Quantitative Assessment of Fluorescein Angiography Leakage via Deep Learning in Pediatric Uveitis: Correlation with Clinical Parameters**。研究目标是用深度学习自动分割儿童葡萄膜炎患者荧光素眼底血管造影（FA）图像中的泄漏区域，并将泄漏面积量化，用于和临床炎症指标做相关性分析。

论文主要分割两类目标：

- 视网膜血管泄漏（retinal vascular leakage）
- 黄斑水肿泄漏（macular edema leakage, ME leakage）

总体流程为：FA 图像输入模型，模型输出泄漏区域 mask，再按像素面积计算泄漏范围，并与玻璃体细胞（VC）、中央黄斑厚度（CMT）、最佳矫正视力（BCVA）等临床指标做统计分析。

## 2. 原论文使用的模型和训练设置

论文没有提出一个全新的网络结构，而是比较了 4 个开源分割架构：

- U-Net
- HRNet
- MsTGANet
- TransUNet

训练细节：

- 输入图像统一预处理后，在模型训练阶段缩放到 `512 x 512`
- 使用五折交叉验证，按患者划分训练集和验证集
- 使用 Dice 作为主要评价指标
- 权重从头初始化，未使用外部预训练权重
- 优化器为 SGD，momentum 为 0.9，weight decay 为 `1e-4`
- 初始学习率为 0.05，训练 100 epoch，batch size 为 8
- 训练平台包含 **NVIDIA GeForce RTX 4060 Laptop GPU** 和 16 GB RAM

主要结果：

- 内部验证中，U-Net 综合表现最好
- U-Net 的视网膜血管泄漏 Dice 约为 0.7241
- U-Net 的黄斑水肿泄漏 Dice 约为 0.7273
- 外部验证中，视网膜血管泄漏泛化相对稳定，最佳平均 Dice 约为 0.7015
- 黄斑水肿泄漏在外部数据上更困难，最佳平均 Dice 约为 0.5918，尤其在 102° 大视野图像上下降明显

临床分析结论：

- VC 和 CMT 与视网膜泄漏最稳定相关
- CMT 和 BCVA 与黄斑水肿泄漏更相关
- 总可见视网膜泄漏面积相对下降约 54.8% 可作为判断炎症趋于安静的候选阈值，AUC 约为 0.905

## 3. 原方案的主要局限

结合论文结果和任务特点，原方案有几个明显改进空间：

- 将图像缩放到 `512 x 512` 会损失小病灶和黄斑区域细节，尤其不利于 102° 大视野图像。
- 黄斑水肿泄漏样本较少，类别不平衡明显。
- FA 泄漏边界具有主观性和模糊性，单纯 Dice loss 难以稳定优化边界。
- 原论文从头训练，没有利用通用视觉预训练权重。
- 原方案只做模型横向比较，对高分辨率 patch、边界细化、FOV 条件建模、半监督等策略利用较少。

## 4. 相关计算机分割论文调研

本节只关注计算机视觉和医学图像分割方法论文，不讨论眼科临床论文。

| 方向 | 代表论文 | 核心启发 | 对 FA 泄漏分割的意义 |
| --- | --- | --- | --- |
| 强医学分割基线 | [nnU-Net](https://www.nature.com/articles/s41592-020-01008-z) | 自动配置预处理、patch、增强、后处理，说明 pipeline 很关键 | 不应只比较网络结构，还要系统优化训练策略 |
| 高分辨率表征 | [HRNet](https://arxiv.org/abs/1908.07919) | 始终保留高分辨率特征 | 适合小病灶、细边界和黄斑区域分割 |
| CNN + Transformer | [TransUNet](https://arxiv.org/abs/2102.04306) | CNN 提取局部纹理，Transformer 建模全局关系 | 适合 FA 图像中广域泄漏与局部细节并存的情况 |
| 轻量 Transformer 分割 | [SegFormer](https://arxiv.org/abs/2105.15203) | 层级 Transformer encoder 和轻量 MLP decoder | 可作为中小规模数据下的替代 backbone |
| 通用 mask decoder | [Mask2Former](https://openaccess.thecvf.com/content/CVPR2022/html/Cheng_Masked-Attention_Mask_Transformer_for_Universal_Image_Segmentation_CVPR_2022_paper.html) | 使用 masked attention 做统一分割 | 对不规则泄漏区域有启发，但直接使用可能偏重 |
| 边界细化 | [PointRend](https://arxiv.org/abs/1912.08193) | 对不确定边界点进行高分辨率 refinement | 适合 FA 泄漏边界模糊、上采样后边缘粗糙的问题 |
| 类别不平衡 loss | [Focal Tversky loss](https://arxiv.org/abs/1810.07842) | 强化小目标、难样本和假阴性控制 | 适合 ME 泄漏样本少、目标面积小的问题 |
| 边界 loss | [Boundary loss](https://arxiv.org/abs/1812.07032) | 用距离变换约束边界 | 适合医生标注边界不稳定的泄漏区域 |
| 通用视觉预训练 | [MAE](https://arxiv.org/abs/2111.06377)、[DINOv2](https://arxiv.org/abs/2304.07193) | 大规模自监督可提供强视觉表征 | 本任务数据太少，不建议从零预训练，但可使用其公开预训练权重 |
| 分割基础模型 | [SAM](https://openaccess.thecvf.com/content/ICCV2023/html/Kirillov_Segment_Anything_ICCV_2023_paper.html)、[MedSAM](https://arxiv.org/abs/2304.12306)、[SAMed](https://arxiv.org/abs/2304.13785) | promptable segmentation，适合辅助标注 | 不建议作为最终全自动模型，但可用于医生修 mask 或伪标签辅助 |

## 5. 数据和算力约束判断

当前约束：

- 数据集最多约 2k 张 FA 图像
- 最多使用一张 RTX 3090，显存通常为 24 GB
- 数据不能继续扩充

因此，不建议在这 2k 张图上从零做 MAE/DINO 这类自监督预训练。原因是：

- 自监督预训练依赖大量、多样化的无标注数据
- 2k 张同模态 FA 图像容易让模型学到设备、亮度和视野偏差
- 从零预训练收益不稳定，甚至可能不如直接使用 ImageNet/DINOv2 预训练权重

更合理的策略是：

- 使用 ImageNet、DINOv2 或 ConvNeXt 公开预训练权重初始化 encoder
- 不从零做大规模自监督
- 如需域适配，只做轻量 warm-up，例如低学习率 masked reconstruction 10 到 30 epoch，但不作为主线依赖

## 6. 推荐方案：FA-LeakFormer-Lite

针对 2k 图像和单张 3090，推荐一个中等规模、稳健优先的方案：

**ImageNet/DINOv2 pretrained ConvNeXt-Tiny encoder + U-Net/FPN decoder + FOV embedding + PointRend 边界细化 + Dice/Focal Tversky/Boundary loss**

### 6.1 输入设计

将单通道 FA 图像构造成 3 通道输入：

- 原始灰度 FA 图
- CLAHE 或 gamma/contrast 增强图
- 高荧光/血管增强图，例如 top-hat、Frangi 或简单高通增强

这样可以让模型同时看到原始信号、局部对比度和泄漏/血管相关结构。

### 6.2 网络结构

推荐结构：

- Encoder：ConvNeXt-Tiny 或 SegFormer-B1，优先使用公开预训练权重
- Decoder：U-Net/FPN 多尺度解码器
- Auxiliary head：可选图像级泄漏存在分类头，用于辅助判断是否存在 ME 泄漏
- Boundary refinement：PointRend 风格细化头，只在不确定边界点上做高分辨率 refinement
- FOV condition：将 55°/102° 视野作为条件 embedding 输入模型，减少不同视野分布差异

输出通道：

- channel 1：视网膜血管泄漏 mask
- channel 2：黄斑水肿泄漏 mask

如果两类标签可能重叠，建议使用 multi-label sigmoid，而不是 softmax。

### 6.3 分辨率和 patch 策略

不建议固定缩放到 `512 x 512` 后直接训练。推荐：

- 全图分支：`768 x 768` 或 `896 x 896`
- 局部高分辨率 patch：围绕黄斑区、后极部和高荧光候选区域采样
- 训练时混合全图和 patch，保证既有全局上下文，也有局部细节
- 推理时使用 sliding window 或 multi-scale inference，再映射回原始图像尺度计算面积

在 RTX 3090 上的可行配置：

- `768 x 768`：batch size 4 到 8，使用 AMP 混合精度
- `1024 x 1024`：batch size 2 到 4，适合最终 fine-tune 或高分辨率 patch

### 6.4 Loss 设计

推荐组合：

```text
Loss = Dice loss + Focal Tversky loss + Boundary loss
```

其中：

- Dice loss 保证区域重叠
- Focal Tversky loss 处理 ME 小目标和类别不平衡
- Boundary loss 约束边界，缓解泄漏边界模糊和标注差异

可对 ME 类别设置更高权重，例如 `1.5` 到 `2.0`，具体通过验证集调参。

### 6.5 数据增强

推荐增强：

- horizontal / vertical flip
- 小角度旋转
- gamma、brightness、contrast、CLAHE
- gaussian blur、motion blur、median blur
- gaussian noise
- random crop 和 high fluorescence crop
- class-balanced sampling，保证每个 batch 中有足够 ME 阳性样本

不建议过强的形变增强，以免破坏 FA 血管结构和泄漏形态。

### 6.6 训练策略

推荐训练流程：

1. 使用预训练 encoder 初始化模型。
2. 先冻结 encoder 前几层训练 decoder 10 到 20 epoch。
3. 解冻全模型，使用较小学习率 fine-tune 150 到 300 epoch。
4. 使用 patient-level 五折交叉验证。
5. 保存每折最优模型，最后使用 ensemble 或选择验证表现最好的模型。

推荐优化器：

- AdamW
- encoder 学习率：`1e-4`
- decoder 学习率：`3e-4`
- weight decay：`1e-4`
- scheduler：cosine annealing 或 poly scheduler

## 7. 评价指标

除了 Dice，建议增加以下指标：

- Dice：区域重叠
- lesion-level recall / F1：是否漏掉小泄漏灶
- boundary F1 或 surface Dice：边界质量
- area MAE / correlation：预测泄漏面积与人工面积的一致性
- 按视野分组的 Dice：分别统计 55° 和 102°
- 按类别分组的 Dice：分别统计视网膜血管泄漏和 ME 泄漏

最终临床量化使用的面积指标，建议在原始图像尺度上计算，避免缩放带来的像素面积偏差。

## 8. 推荐实验优先级

第一阶段，建立强基线：

- ConvNeXt-Tiny U-Net/FPN
- ImageNet 或 DINOv2 预训练
- `768 x 768`
- Dice + Focal Tversky
- patient-level 五折交叉验证

第二阶段，提高小病灶和边界：

- 加入 Boundary loss
- 加入 high-res patch
- 加入 PointRend refinement
- 加入 ME class-balanced sampling

第三阶段，提高泛化：

- 加入 FOV embedding
- 做 55°/102° 分组评估
- 做 test-time augmentation
- 尝试轻量 ensemble

第四阶段，辅助标注和伪标签：

- 使用 SAM/MedSAM 辅助医生修 mask
- 对低置信边界区域进行主动复核
- 不建议将 SAM 作为最终全自动分割模型

## 9. 最终建议

在当前条件下，最推荐的主线不是“大模型从零预训练”，而是：

```text
预训练中小型 encoder
+ 高分辨率训练
+ 小目标友好的 loss
+ 边界细化
+ FOV 条件建模
+ 严格 patient-level 验证
```

具体落地模型：

```text
FA-LeakFormer-Lite
= ConvNeXt-Tiny pretrained encoder
+ U-Net/FPN decoder
+ FOV embedding
+ PointRend boundary refinement
+ Dice/FocalTversky/Boundary loss
```

这个方案比原论文单纯比较 U-Net、HRNet、MsTGANet、TransUNet 更贴近本任务的真实难点：数据量有限、小病灶、边界模糊、ME 类别不平衡、不同视野泛化差。对于 2k 张图像和单张 RTX 3090，它也比从零训练大型 Transformer 或 SAM 更现实。
