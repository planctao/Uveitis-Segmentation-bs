# 相关方案调研：可讲故事的小创新路线

日期：2026-06-11

## 1. 调研结论

原论文的主要叙事是“比较 U-Net、HRNet、MsTGANet、TransUNet 四个分割网络”。这个叙事偏横向模型比较，创新性比较弱。如果继续只换成 ConvNeXt、SegFormer、Swin-Unet，本质上仍然像是在换 backbone。

更适合本毕设的路线是：不要把创新放在“大模型结构”，而是放在 FA 泄漏任务的特性上。

FA 泄漏分割有几个很适合讲故事的特点：

- 泄漏区域是高荧光、边界模糊、形态弥散的病灶，不是普通自然图像里的清晰物体。
- 小病灶和黄斑水肿泄漏容易被背景淹没，类别不平衡明显。
- 原论文把图缩到 `512 x 512`，会损失小区域和边界细节；当前数据本身是 `768 x 768`，可以利用。
- 最终临床用途不是单纯 mask 好看，而是泄漏面积量化，所以面积误差、相关性和可解释可视化都可以成为论文叙事的一部分。
- mask 标签值目前存在 `{1,2,3}` 等语义未完全确认的情况，主线要保留二值总泄漏兜底。

因此，推荐把论文故事写成：

```text
针对 FA 泄漏区域高荧光、边界模糊、小目标不均衡和临床面积量化需求，
设计一个轻量级的荧光先验引导 + 边界不确定性建模 + 面积量化约束的分割框架。
```

这比“换 backbone”更像一个方法。

## 2. 相关工作启发

### 2.1 FA 泄漏分割不是特别容易

儿童葡萄膜炎 FA 泄漏原论文使用 U-Net、HRNet、MsTGANet 和 TransUNet 做 retinal vascular leakage 与 ME leakage 分割，说明这个任务已有直接 baseline，但论文主体还是模型比较。

另一篇 retinal vasculitis FA 泄漏/闭塞分割工作使用 DeeplabV3+、UNet++、UNet 等二值分割模型，vascular leakage Dice 约 `0.6279`，occlusion Dice 约 `0.6992`，也说明 FA 泄漏边界和标注本身并不轻松。

CNV leakage 的 FA 分割工作在更大数据上做了自动泄漏面积分割，并强调了 uncertainty estimation 对临床采用和人工复核的重要性。这一点很适合迁移到葡萄膜炎泄漏：模型不仅输出 mask，还输出哪里不确定。

### 2.2 pipeline 和训练策略本身可以成为贡献

nnU-Net 的核心启发不是“换一个新网络”，而是自动配置预处理、patch、增强、训练和后处理。对本任务来说，可以把贡献放在适合 FA 泄漏的 pipeline：高分辨率、荧光先验、patch/边界采样、面积量化，而不是堆新 backbone。

### 2.3 小目标和类别不平衡 loss 有文献支撑

Focal Tversky loss 专门面向高度不平衡和小 ROI 分割，比单纯 Dice 更适合作为 ME 泄漏和小泄漏灶的训练目标。

Unified Focal loss 进一步说明 Dice/CE/focal 类损失的选择会显著影响类别不平衡医学分割的收敛和召回-精度平衡。本文不一定要实现复杂的 Unified Focal，但可以引用其动机，使用 Dice + Focal Tversky 或 Dice + Focal BCE 作为稳妥方案。

### 2.4 边界和不确定性是医学分割常见突破口

PointRend 的思想是对不确定点做高分辨率 refinement，而不是整张图无差别增加计算量。FA 泄漏边界模糊、上采样后容易粗糙，适合借鉴“只细化不确定区域”的思路。

医学图像分割里也有 boundary uncertainty / soft boundary loss 的工作，把边界附近视为不确定区域，不强迫模型在模糊边界上学习过硬的 0/1 标签。FA 泄漏边界主观性强，这个故事很贴合。

### 2.5 SAM/MedSAM 可以作为辅助，不建议当主线

MedSAM 说明医学分割基础模型有潜力，但相关综述也指出 SAM 直接用于医学图像时，对弱边界、小目标、低对比、形状不规则的目标表现不稳定。FA 泄漏恰好就是弱边界、小目标、形态弥散任务，所以不建议把 SAM 作为最终全自动主模型。

更稳的用法是：把 MedSAM/SAM 当作辅助标注、候选区域生成或边界复核工具，而不是主线模型。

## 3. 候选方法一：FA-UGRNet-Lite

推荐主线名称：

```text
FA-UGRNet-Lite
= Fluorescence-prior Guided Uncertainty Refinement Network
```

中文可以叫：

```text
荧光先验引导的边界不确定性细化网络
```

这个名字听起来像方法，但实现不需要很重。

### 3.1 核心故事

FA 泄漏不是普通分割对象，而是由荧光素渗漏形成的高荧光弥散区域。它有两个直接问题：

- 亮度和对比度提示很强，但不同患者/拍摄相位/视野下亮度分布变化大。
- 泄漏边界并没有清晰物理边缘，人工标注也会有主观差异。

所以方法不只是换 encoder，而是显式利用两个任务先验：

```text
荧光强度先验：告诉模型哪里更像候选泄漏区域
边界不确定性先验：告诉模型边界附近不要按普通硬标签粗暴学习
```

### 3.2 结构设计

基础网络：

```text
ConvNeXt-Tiny / ResNet34 pretrained encoder
+ U-Net/FPN decoder
+ segmentation head
+ boundary/uncertainty auxiliary head
+ optional area regression head
```

输入不直接使用原 RGB，而是构造 FA-aware 多通道输入：

```text
channel 1: 原始灰度图
channel 2: CLAHE 或 gamma 增强图
channel 3: top-hat / high-pass 高荧光候选图
channel 4: optional vesselness / edge map
```

如果想保持 3 通道兼容 ImageNet 权重：

```text
3-channel = [gray, CLAHE, high-fluorescence prior]
```

这样就不是“把 RGB 丢进模型”，而是把 FA 泄漏的成像机制编码进输入。

### 3.3 训练目标

主分割 loss：

```text
L_seg = Dice loss + Focal Tversky loss
```

边界辅助：

```text
boundary_gt = dilate(mask) - erode(mask)
L_boundary = BCE(boundary_pred, boundary_gt)
```

面积量化辅助，可选：

```text
area_gt = foreground_pixels / valid_retina_pixels
area_pred = mean(sigmoid(pred_mask))
L_area = SmoothL1(log(area_pred + eps), log(area_gt + eps))
```

总 loss：

```text
L = L_seg + 0.2 * L_boundary + 0.1 * L_area
```

如果想更轻：

```text
L = Dice + Focal Tversky + boundary-weighted BCE
```

边界不确定性不一定要做复杂 Bayesian 模型。可以把边界带作为 uncertain band，在该区域降低硬标签惩罚或单独监督 boundary head，这已经足够讲故事。

### 3.4 推理阶段

输出：

- 泄漏 mask
- boundary/uncertainty map
- 面积比例
- overlay 可视化

可以加入低成本 TTA：

```text
原图
水平翻转
垂直翻转
水平+垂直翻转
```

预测均值作为 mask，预测方差作为 uncertainty。这样论文里可以展示“模型对模糊边界更不确定”，很有可解释性。

### 3.5 实验表

建议做消融：

```text
B0: U-Net/ResNet34, RGB/gray, BCE+Dice
B1: B0 + 768x768 高分辨率
B2: B1 + FA-aware input prior
B3: B2 + Focal Tversky
B4: B3 + boundary/uncertainty head
B5: B4 + area auxiliary 或 TTA uncertainty
```

论文贡献可以写成：

```text
1. 提出一种面向 FA 泄漏的荧光先验输入构造方式，利用高荧光候选图增强小泄漏区域感知。
2. 引入边界不确定性辅助监督，缓解泄漏边界模糊和人工标注主观性。
3. 加入面积量化约束，使模型优化目标与临床泄漏面积评估更一致。
```

### 3.6 可行性判断

这个方案适合当前数据和算力：

- 不需要额外标注。
- 不需要训练大模型。
- 主要是数据预处理、loss 和 auxiliary head。
- 单张 3090/4090 可用 `768 x 768`、batch size `4` 起步。
- 即使多标签语义没确认，也能先做 `mask > 0` 二值总泄漏。

这是最推荐的主线。

## 4. 候选方法二：Quantification-Aware FA Leakage Segmentation

中文：

```text
面积量化感知的 FA 泄漏分割
```

### 4.1 核心故事

原任务最终不是单纯分割，而是用泄漏面积和临床炎症指标做相关性分析。因此模型不应该只优化 Dice，还应该关心面积预测是否准确。

方法：

```text
segmentation branch: 输出 mask
classification branch: 判断是否存在泄漏/ME 泄漏
area branch: 回归泄漏面积比例
```

所有监督都可以从 mask 自动生成：

- 是否存在泄漏：`area_gt > 0`
- 泄漏面积比例：`foreground_pixels / valid_pixels`
- 小/中/大泄漏分级：按面积分位数切三档

### 4.2 创新点

这个方案的创新点不在网络，而在任务建模：

```text
从像素重叠优化，扩展到临床量化目标优化。
```

论文可讲：

- Dice 对边界很敏感，但临床更关心面积趋势。
- 面积辅助约束让模型对漏检小病灶和过分扩张边界更敏感。
- 面积相关性、Bland-Altman、MAE 可以成为很漂亮的结果图。

### 4.3 推荐用法

这个方向非常适合和方法一合并：

```text
FA-UGRNet-Lite = 荧光先验 + 边界不确定性 + 面积量化辅助
```

如果时间紧，面积辅助比 PointRend 更容易实现。

## 5. 候选方法三：Coarse-to-Fine High-Fluorescence Patch Refinement

中文：

```text
高荧光候选区域引导的粗到细分割
```

### 5.1 核心故事

FA 泄漏小病灶在全图中占比低，模型容易被背景主导。先用全图模型获得粗预测，再围绕候选高荧光区域或不确定区域做局部 patch refinement，可以提高小区域细节。

训练时：

```text
50% full image
25% foreground/boundary patch
25% high-fluorescence candidate patch
```

推理时：

```text
full image prediction
-> 找到高不确定/高荧光 ROI
-> 对 ROI 做 patch prediction
-> 融合回全图
```

### 5.2 适合怎么做

不要一开始做复杂两阶段网络。先做 patch sampler：

- 根据 mask 前景采样正样本 patch。
- 根据 high-fluorescence prior 采样疑似泄漏 patch。
- 保留一半全图训练，避免只有局部没有上下文。

等 baseline 稳了，再做真正两阶段推理。

### 5.3 风险

- patch 融合代码比单图训练复杂。
- 如果标签边界很粗，patch 可能只提升局部 Dice，不一定提升面积相关性。
- 对毕业来说，这是加分项，不建议放在第一优先级。

## 6. 候选方法四：FOV/Quality Conditioned Segmentation

中文：

```text
视野与图像质量条件化的泄漏分割
```

### 6.1 核心故事

原论文提到不同视野，尤其 102° 大视野图像上 ME leakage 泛化更困难。若数据里有 55°/102° 元信息，可以把 FOV 作为条件输入。

如果没有显式 FOV 标签，也可以自动计算图像统计特征：

- 有效视网膜区域面积比例。
- 黑边/背景比例。
- 平均亮度和对比度。
- 高荧光像素比例。
- 中心/外周亮度分布。

把这些特征输入一个小 MLP，生成 FiLM 参数或直接拼到 decoder bottleneck。

### 6.2 优点

这个故事比“换模型”更贴近原论文痛点：

```text
解决不同视野和拍摄条件导致的域偏移。
```

### 6.3 风险

- 当前数据目录没有显式 FOV 标签。
- 自动 FOV 估计可能需要额外调试。
- 如果五折里 FOV 分布很均匀，提升可能不明显。

建议作为第三优先级，不要作为主创新。

## 7. 候选方法五：SAM/MedSAM Assisted Boundary Review

中文：

```text
基于 MedSAM 的边界辅助复核
```

### 7.1 核心故事

SAM/MedSAM 不作为主模型，而作为辅助工具：

- 用训练好的模型预测粗 mask。
- 根据粗 mask 生成 box/point prompt。
- 让 MedSAM 生成候选 mask。
- 只在模型高不确定区域或边界区域做比较/融合/人工复核。

### 7.2 为什么不推荐作为主线

FA 泄漏弱边界、小目标、低对比、形态不规则，正是 SAM 零样本医学分割容易不稳定的场景。直接把 SAM 当主模型，结果未必好，调参成本也高。

### 7.3 适合放在哪里

可以放在论文讨论或附录：

```text
基础模型可作为辅助标注工具，但当前任务仍需要面向 FA 泄漏特性的专用轻量模型。
```

## 8. 最推荐的最终方案

综合创新性、实现难度、3090/4090 算力和毕业可控性，推荐最终主线：

```text
FA-UGRNet-Lite:
  1. 768x768 预训练轻量 encoder-decoder baseline
  2. FA-aware fluorescence prior input
  3. Dice + Focal Tversky 小目标损失
  4. boundary uncertainty auxiliary head
  5. optional area quantification auxiliary loss
```

最小版本：

```text
gray + CLAHE + high-fluorescence prior
-> ConvNeXt-Tiny/ResNet34 U-Net
-> Dice + Focal Tversky
-> boundary head
-> 五折验证
```

完整版本：

```text
最小版本
+ area regression head
+ TTA uncertainty map
+ high-fluorescence patch sampling
```

## 10. 实验设计建议

### 10.1 主实验

五折交叉验证：

```text
train: 4 folds
val/test: 1 fold
repeat for f1..f5
```

先用二值总泄漏：

```text
foreground = mask > 0
```

标签语义确认后，再做：

```text
retinal vascular leakage
ME leakage
other/ignore
```

### 10.2 消融实验

建议表格：

```text
Method                               Dice   IoU   Recall   Area MAE   Area Corr
U-Net 512                            ...
U-Net 768                            ...
Pretrained U-Net/FPN 768             ...
+ fluorescence prior                 ...
+ Focal Tversky                      ...
+ boundary uncertainty head          ...
+ area auxiliary / TTA uncertainty   ...
```

不要一次堆太多。只要每一步能解释，就可以讲故事。

### 10.3 可视化

必须做：

- 原图
- GT
- baseline 预测
- proposed 预测
- uncertainty/boundary map
- false positive / false negative overlay

最适合展示的病例：

- 小泄漏灶。
- 黄斑区域泄漏。
- 边界模糊大面积泄漏。
- baseline 漏检但 proposed 找到的样本。
- proposed 仍失败的样本，用于局限分析。

## 11. 开题/论文可用贡献写法

可以写三点贡献：

```text
1. 针对 FA 泄漏高荧光弥散成像特点，提出一种无需额外标注的荧光先验输入构造策略，使模型显式感知潜在泄漏候选区域。

2. 针对泄漏边界模糊和小目标不均衡问题，引入边界不确定性辅助监督与 Focal Tversky 优化目标，提高模型对模糊边界和小泄漏灶的鲁棒性。

3. 针对临床炎症评估中的泄漏面积量化需求，加入面积一致性约束，并从 Dice、IoU、面积误差和面积相关性多个维度评估模型。
```

这三点听起来像一个完整方法，不是简单换模型。

## 12. 优先级排序

最值得做：

```text
1. fluorescence prior input
2. Dice + Focal Tversky
3. boundary head / boundary weighted loss
4. area auxiliary loss
```

可以做但不急：

```text
5. high-fluorescence patch sampling
6. TTA uncertainty
7. FOV/quality conditioning
```

不建议作为主线：

```text
8. SAM/MedSAM 全量 fine-tune
9. 从零自监督预训练
10. 大型 transformer 结构堆叠
```
